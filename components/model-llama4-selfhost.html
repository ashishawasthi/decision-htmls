Llama 4 (self-host) in agentic systems

An open-weight model you run yourself on a GPU fleet, so data never leaves your boundary and there is no per-token API bill. You pay for the fleet instead. The design rule that follows: choose it for residency or control, and only when steady volume keeps the GPUs busy enough to beat per-token pricing.

Spec at a glance

ProviderOpen weights (Meta Llama), served on GKE with vLLM
ModalityText (multimodal variants available depending on checkpoint)
Context windowCheckpoint-dependent
Token priceNo per-token API charge; cost is the GPU fleet (accelerator, precision, utilisation)
Integrated search toolNo. Live web grounding is unavailable, so adding a web source raises a warning in the linter
LatencyDepends on the fleet. Time to first token around 500 ms (illustrative) at the modelled throughput
Best role in this toolBoth roles, locked in by On-prem deployment

When to pick it

Good fit

  • On-prem or strict data-residency requirements where prompts cannot leave the boundary.
  • High, steady volume where an amortised GPU fleet undercuts per-token API pricing.
  • Full control over weights, versioning, and the serving stack.

Avoid

  • Spiky or low volume: idle GPUs waste the fixed fleet cost.
  • Teams without the MLOps to run and tune a serving cluster.
  • Designs that depend on a managed web-search tool for live grounding.

How it maps to this tool

  • Locked in by On-prem deployment. Choosing On-prem locks both the reasoning and fast roles to this model. Token cost drops to zero and the self-host GPU fleet cost model takes over (accelerator class, quantisation, and utilisation drive the node count).
  • Integrated search tool: no. Selecting it as the reasoning model with a live web source raises a warning, and live web grounding will not run.
  • Self-host knobs apply. The Accelerator, Quantisation, GPU pricing, and Utilisation controls size the fleet and set the inference cost.
See also. The serving layer is covered in the vLLM on GKE component page, which details how throughput, precision, and accelerator choice size the fleet.
