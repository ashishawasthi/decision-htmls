Gemini 3.1 Flash-Lite in agentic systems

The cheapest and fastest model in this lineup. It is built for high-volume, latency-sensitive work: the fast lane behind smart routing, autocomplete, classification, and extraction. The design rule that follows: send the simple, high-frequency traffic here and reserve a reasoning model for the hard steps.

Spec at a glance

ProviderGoogle (Gemini API / Agent Platform)
ModalityMultimodal input (text, image, audio, video), text output
Context windowAbout 1M tokens
Integrated search toolYes, Google Search grounding. Drives live web grounding when this is the reasoning model
Context cacheSupported, the cheapest cache read in this lineup
LatencyFastest tier here
Best role in this toolFast / lookup model (the default)

When to pick it

Good fit

  • The fast lane in smart routing: the share set by % to fast model runs here.
  • High-volume Q&A and sub-second SLOs where cost per call and time to first token dominate.
  • Cheap, well-scoped steps: classification, extraction, query rewriting, autocomplete suggestions.

Avoid

  • The hardest multi-step reasoning or planning: route those to a reasoning model like Gemini 3 Pro or Claude Opus 4.8.
  • Work where answer quality outranks cost and speed: step up to Gemini 3.5 Flash or a reasoning model.
  • As the only model in a system that also needs deep synthesis or judging.

How it maps to this tool

  • Default Fast / lookup model. Smart routing sends the % to fast model share here; its low latency and low output cost pull the blended Generation latency and the output cost down.
  • Integrated search tool: yes. It carries Google Search grounding, so when set as the reasoning model a live web source runs live grounding.
  • Context cache. Reused input is the cheapest cache read here, so a high reuse share on the Context cache lever barely adds cost.
Routing is where it pays off. Raise % to fast model and the Generation row in the latency breakdown drops, because more of the blend runs on Flash-Lite. Drop smart routing and every call falls back to the reasoning model, so both latency and cost rise.
