Gemini 3.1 Flash-Lite in agentic systems

The cheapest and fastest model in this lineup. It is built for high-volume, latency-sensitive work: the fast lane behind smart routing, autocomplete, classification, and extraction. The design rule that follows: send the simple, high-frequency traffic here and reserve a reasoning model for the hard steps.

When to pick it

Good fit

  • The fast lane in smart routing: the share set by % to fast model runs here.
  • High-volume Q&A and sub-second SLOs where cost per call and time to first token dominate.
  • Cheap, well-scoped steps: classification, extraction, query rewriting, autocomplete suggestions.

Avoid

  • The hardest multi-step reasoning or planning: route those to a reasoning model.
  • Work where answer quality outranks cost and speed.
  • As the only model in a system that also needs deep synthesis or judging.

How it maps to this tool

  • Default Fast / lookup model. Smart routing sends the % to fast model share here; its low time to first token and $1.50 output price pull the blended Generation latency and the output cost down.
  • Integrated search tool: yes. It carries Google Search grounding, so when set as the reasoning model a live web source runs live grounding, billed about $14 per 1k searches.
  • Context cache. Reused input bills at $0.025 per 1M, the cheapest cache read here, so a high reuse share on the Context cache lever barely adds cost.
Routing is where it pays off. Raise % to fast model and the Generation row in the latency breakdown drops, because more of the blend runs on Flash-Lite. Drop smart routing and every call falls back to the reasoning model, so both latency and cost rise.
