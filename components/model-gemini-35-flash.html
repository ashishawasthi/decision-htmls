Gemini 3.5 Flash in agentic systems

The balanced default for reasoning: strong enough for agentic planning and grounded answers, while staying far cheaper and faster than a frontier model. The design rule that follows: make it the reasoning model for most copilots and let Flash-Lite take the simple, high-volume share.

When to pick it

Good fit

  • The default reasoning model for an agentic copilot: balanced cost, quality, and speed.
  • Grounded RAG answers and multi-step plans inside an agentic latency budget.
  • Multimodal grounding where image or document understanding matters.

Avoid

  • Ultra-cheap, very high-volume calls: send those to Flash-Lite instead.
  • The absolute hardest reasoning or long-context synthesis: step up to Gemini 3 Pro or Claude Opus.

How it maps to this tool

  • Default Reasoning model. With smart routing on it handles the non-fast share of traffic, so it sets the p95 tail of the Generation stage.
  • Integrated search tool: yes. Google Search grounding, so adding a web source runs live web grounding without a warning, billed about $14 per 1k searches.
  • Context cache. Reused input bills at $0.15 per 1M, so the Context cache lever discounts the shared system prompt and retrieved context.
